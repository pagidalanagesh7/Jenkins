# 🎯 Jenkins Interview Questions & Answers

Welcome to the Jenkins Day 1 interview preparation guide.

This document contains beginner-friendly interview questions with simple and clear answers.

---

# Q1. What is Jenkins?

### Answer

Jenkins is an open-source automation server used to automate software development tasks such as building, testing, and deploying applications.

It supports Continuous Integration (CI) and Continuous Delivery (CD), helping teams deliver software faster and more reliably.

---

# Q2. Why is Jenkins used?

### Answer

Jenkins automates repetitive tasks in the software delivery process.

It helps:

- Automate builds
- Run automated tests
- Deploy applications
- Reduce manual errors
- Speed up software delivery

---

# Q3. What is Continuous Integration (CI)?

### Answer

Continuous Integration (CI) is the practice of frequently merging code changes into a shared repository.

Whenever code is pushed, Jenkins automatically builds and tests the application.

This helps detect bugs early and improves code quality.

---

# Q4. What is Continuous Delivery (CD)?

### Answer

Continuous Delivery (CD) is the practice of automatically preparing applications for deployment after successful testing.

It ensures software is always ready for release.

---

# Q5. What is the difference between CI and CD?

| Continuous Integration | Continuous Delivery |
|-------------------------|--------------------|
| Focuses on Build & Test | Focuses on Release & Deploy |
| Detects Bugs Early | Automates Software Delivery |
| Developers Merge Code Frequently | Applications are Ready for Deployment |

---

# Q6. What can Jenkins automate?

### Answer

Jenkins can automate:

- Building applications
- Running tests
- Static code analysis
- Docker image creation
- Deployments
- Notifications
- Scheduled jobs

---

# Q7. Why is Jenkins called an Automation Server?

### Answer

Because Jenkins automates repetitive software development and deployment tasks without manual intervention.

---

# Q8. What are Jenkins Plugins?

### Answer

Plugins extend Jenkins functionality.

Examples:

- Git Plugin
- Docker Plugin
- Kubernetes Plugin
- Maven Plugin
- Pipeline Plugin

Jenkins has thousands of community-supported plugins.

---

# Q9. Why do enterprises still use Jenkins?

### Answer

Because Jenkins is:

- Open Source
- Highly Customizable
- Plugin Rich
- Supports Every Programming Language
- Integrates with Almost Every DevOps Tool
